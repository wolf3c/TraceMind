# TraceMind Customer Messaging

本文档用于沉淀 TraceMind 对外客户文案。后续根据产品发展、客户外联回复、试点转化和真实反馈持续迭代。

## 当前核心文案

```text
TraceMind 的三个核心优势：

1. AI 自动埋点
不用设计复杂埋点、维护看板、搭数据体系，AI coding agent 自动采集并整理用户行为。

2. AI 分析数据
运营效果、转化问题、新功能使用情况，直接问 AI，不用自己翻看板。

3. AI 迭代闭环
从发现问题、辅助改动，到验证效果，回到 AI coding workflow 里持续完成。
```

## 一句话版本

```text
AI 自动埋点，AI 分析数据，AI 驱动产品迭代闭环。
```

## 客户视角表达原则

- 少讲技术架构，多讲客户少做什么、多知道什么、能更快完成什么。
- 优先强调“不用重建 analytics 体系”和“直接问 AI”。
- 避免过度承诺“全自动替客户做产品决策”，保留客户判断权。
- 公开文案要简洁，私信文案再补充具体试点承诺。

## 使用场景

- X / Twitter、即刻、V2EX 等公开帖。
- 私信、邮件、官网首屏和产品介绍页。
- 给早期 design partner 的试点说明。
- Product Hunt、Hacker News、MCP 目录等渠道的短介绍。

## 后续优化依据

每次外联或发布后，记录以下信息：

- 渠道和目标客户类型。
- 使用的文案版本。
- 回复率、点击率、约访率或试点转化。
- 客户听不懂、质疑或最感兴趣的表达。
- 是否需要调整优势排序、措辞或承诺边界。
