产品更新

这里是产品更新的独立详情页。每次更新按时间倒序排列，先看摘要标题，再看详细要点。

2026-05-29

Agent 健康摘要会先读取每日健康报告，再按语义事件、路径和原始证据定位变化原因

  • 分析入口从“先问一个开放问题”调整为“先读健康摘要，再决定是否下钻”。
  • 更适合排查活跃下降、功能使用波动、采集异常和推广后的行为变化。
  • 减少 Agent 在无上下文状态下猜查询路径的情况。
2026-05-22

推广或运营动作后，可以按小时观察活跃用户、事件量和趋势变化

  • 项目健康卡片新增小时级趋势表达，用于观察一天内不同时段的活跃变化。
  • 更适合看推广投放、社媒发布、社群运营和新功能上线后的即时反馈。
  • 帮助判断变化从哪个小时开始改善或下滑。
2026-05-17

产品页补充了 TraceMind 如何把用户行为、Agent 分析和确认后的反馈连接成循环

  • 默认分析路径仍然只读，只有在开发者确认后才提交结构化反馈。
  • 页面展示了产品增长循环和 TraceMind 智能进化循环。
  • 避免把反馈机制放在核心 Auto Capture、AI-readable evidence 和 MCP 分析价值之前。
